Writing Style Rules:

1. Use simple, clear, natural English.
2. Use common everyday words.
3. Keep sentences short and easy to understand.
4. Use simple Indian English that is natural and meaningful.
5. Avoid rare, poetic, literary, or overly advanced words.
6. Do not write like an art or game development textbook.
7. Avoid technical art terms unless they are necessary.
8. Explain visual ideas so that a normal person can understand them quickly.
9. Keep every field useful and meaningful.
10. Avoid unnecessary details.
11. Be visually descriptive without writing long paragraphs.
12. Do not repeat the same visual idea in different fields.

Visual Design Rules:

1. Use the Game Plan to understand the main game idea, genre, setting, and objective.
2. Use the Story to understand the emotional journey, major events, conflict, climax, and ending.
3. Use the Characters to understand each important character's role, personality, strengths, weaknesses, and background.
4. Use the Levels to understand the important locations and how the world changes during the game.
5. Use the Gameplay to understand what the player does and which actions need clear visual feedback.
6. Keep all visual decisions connected to the same game world.
7. Every visual choice must fit the existing game.
8. Do not add random visual themes that do not fit the game.
9. Do not make every game dark, realistic, or cinematic by default.
10. Choose the visual direction based on the actual game concept.
11. Keep the visual style practical and consistent.
12. The visual design should support gameplay clarity.
13. Important gameplay objects must be easy for the player to notice.
14. Dangerous elements must look different from safe elements.
15. Important story moments should have stronger visual presentation.
16. Prefer a few strong visual ideas over many unrelated ideas.

Visual Overview Rules:

1. art_style must clearly describe the main visual style of the game.
2. Choose a style that fits the genre and gameplay.
3. Do not automatically choose photorealistic graphics.
4. visual_identity must explain what makes the game visually recognisable.
5. player_visual_experience must explain what the player should feel while looking at and moving through the game world.
6. Keep each field focused on a different purpose.

Color Direction Rules:

1. Create between 3 and 6 primary colors.
2. Create between 1 and 4 accent colors.
3. Use simple colour names.
4. Colours must fit the setting, mood, story, and gameplay.
5. color_usage must explain where and why the main colours are used.
6. Important gameplay elements should have clear colour differences.
7. Do not use too many unrelated colours.
8. Do not make every game use purple and blue unless the game actually needs them.

Lighting and Atmosphere Rules:

1. lighting_style must explain the main lighting direction.
2. atmosphere must explain the overall feeling of the world.
3. Create between 2 and 5 weather or environment effects when they fit the game.
4. Do not add weather effects when they do not make sense.
5. Lighting should change when important story or level events require it.
6. Keep dangerous, peaceful, mysterious, and important areas visually clear.

Environment Design Rules:

1. overall_direction must explain the common visual rules followed by the game world.
2. Use important locations that already exist in the Levels or Story.
3. Create visual direction for between 3 and 6 important locations.
4. Do not invent unrelated locations.
5. Use existing location names exactly when possible.
6. visual_appearance must explain what the location looks like.
7. mood must explain how the location should feel.
8. Create between 2 and 5 important_visual_details for each location.
9. Visual details should include meaningful things the player can notice.
10. Important locations should feel connected to the same world while still looking different from each other.

Character Visual Direction Rules:

1. Include only important existing characters.
2. Use existing character names exactly as given.
3. Do not rename characters.
4. Do not change their existing roles or personalities.
5. The visual design must reflect each character's role and personality.
6. visual_style must explain the overall look of the character.
7. clothing_and_gear must fit the character and game world.
8. Create between 2 and 5 distinctive_features for each character.
9. visual_role must explain how the player can quickly recognise the character's role through appearance.
10. Do not add random weapons, powers, armour, or equipment that do not exist in the game.

Enemy and Creature Design Rules:

1. Use this section only for enemies, creatures, monsters, or hostile forces that already fit the game.
2. If the game has no traditional enemies or creatures, design the visual style of its main threats or dangers instead.
3. overall_style must explain the common visual identity of threats.
4. Create between 2 and 5 short design_rules.
5. visual_threat_progression must explain how threats look more dangerous as the game progresses.
6. Do not create unrelated enemy groups.
7. Do not change the story by inventing major new villains.

Visual Effects Rules:

1. Create between 3 and 6 important visual effects.
2. Effects must connect to actual gameplay, story events, abilities, dangers, or world systems.
3. effect_name must be short and clear.
4. appearance must explain what the effect looks like.
5. used_for must explain when the player sees it.
6. Visual effects must help the player understand what is happening.
7. Avoid effects that are only decorative and have no clear purpose.
8. Do not overload the game with too many effects.

Camera and Presentation Rules:

1. camera_style must fit the existing gameplay.
2. Do not assume first-person, third-person, top-down, side view, or another camera unless it fits the game concept.
3. exploration_presentation must explain how the world is shown during normal play.
4. action_presentation must explain how important gameplay moments are shown.
5. story_presentation must explain how story moments are visually presented.
6. Keep camera ideas practical.
7. Do not use excessive camera shaking or cinematic effects that make gameplay hard to understand.

UI and HUD Style Rules:

1. The UI must visually fit the game world.
2. overall_style must explain the common look of the interface.
3. Create between 3 and 7 important hud_elements.
4. Include only HUD elements that the gameplay actually needs.
5. Do not add health bars, maps, ammo counters, or other elements unless the game needs them.
6. menu_style must explain the visual appearance of menus.
7. feedback_style must explain how the game visually shows success, danger, interaction, progress, or failure.
8. The UI should remain easy to read.
9. Do not sacrifice usability only to make the UI look stylish.

Consistency Rules:

1. All sections must describe the same visual identity.
2. The colour direction must match the environment and character design.
3. Character designs must fit the world.
4. Visual effects must match the art style.
5. UI and HUD must match the game's visual identity.
6. Camera presentation must support the gameplay mechanics.
7. The visual design should become more intense or meaningful as the player moves through the existing level journey.
8. Do not copy the same description into multiple fields.

Important:

1. Keep all visual systems connected to the same game world.
2. Use existing character names exactly as given.
3. Use existing important location names exactly when possible.
4. Do not rename important characters or locations.
5. Do not change the number or order of existing levels.
6. Do not create new story events.
7. Do not create new gameplay mechanics.
8. The output should feel like one complete visual direction for one game.
9. Prefer a few strong visual ideas over many weak ideas.
10. Make the visual design specific to the generated game, not generic advice.
